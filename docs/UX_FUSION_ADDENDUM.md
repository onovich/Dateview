
# 📌 Dateview 任务栏融合增强层（UX Fusion Addendum）

> 本文档为 Dateview 既有架构与开发计划的扩展模块，定义“系统级体验融合层”，用于指导 Codex 在 UI/交互层实现更接近 Windows 原生体验的行为模型。

---

# 🧠 一、目标升级定义

在已有“托盘日历工具”的基础上，引入第三层体验目标：

## 🎯 UX Fusion Layer 目标

- 让 Dateview 在视觉与行为上“接近 Windows 原生日历”
- 提升任务栏交互的系统一致性
- 降低用户对“第三方应用”的感知
- 强化“任务栏扩展能力”而非“独立应用窗口”

---

# 🪟 二、系统级视觉融合规范

## 1. Windows 11 风格强一致性

必须统一以下视觉标准：

- 圆角：8–12px
- 背景：优先 Mica，其次 Acrylic
- 阴影：弱阴影（接近系统面板）
- 动画：120–200ms ease-out
- 字体：Segoe UI Variable

---

## 2. “任务栏延伸”视觉错觉

窗口必须呈现“从任务栏弹出”的效果：

- 初始 opacity = 0
- 初始 transform = translateY(+8~10px)
- 动画结束位置必须紧贴任务栏边缘

关闭动画必须反向收回任务栏区域

---

# 🧭 三、任务栏吸附与定位规则

## 1. Anchor 计算规则

- 基于鼠标点击点确定屏幕
- 自动识别任务栏所在边（bottom/left/top/right）
- 以任务栏边缘为 anchor

---

## 2. Window 定位规则

- window 必须贴合任务栏边缘
- 保持 2–6px 视觉贴边误差（模拟系统层级感）
- 禁止居中弹窗行为

---

## 3. 多屏策略

- 点击在哪个屏幕 → 就在哪个屏幕弹出
- 禁止默认主屏策略

---

# 🖱️ 四、交互融合状态机

## 状态模型

Hidden → TrayCollapsed → CalendarOpen → TrayCollapsed

---

## 行为规则

### 1. 托盘点击
- 打开 CalendarOpen
- 若已打开 → 关闭

### 2. 外部点击
- 立即收起（无延迟）

### 3. ESC
- 强制收起

### 4. 焦点丢失
- 若鼠标不在窗口内 → 收起

---

# 🎬 五、动画系统规范

## 1. 打开动画

- duration: 140–180ms
- easing: cubic-out
- fade + translateY(-8px → 0)

---

## 2. 关闭动画

- duration: 120–140ms
- easing: cubic-in
- 收回任务栏方向

---

# 🧊 六、系统一致性约束

必须满足：

- [ ] 不出现在 Alt+Tab
- [ ] 不显示任务栏窗口按钮
- [ ] DPI PerMonitorV2 支持
- [ ] 多屏缩放正确
- [ ] 无自定义 UI 装饰边框
- [ ] 无非系统级动画风格

---

# ⚠️ 七、禁止行为

严禁以下实现方式：

- hook Windows Shell
- 修改 Explorer 行为
- 注入任务栏 UI
- 自定义系统时钟区域
- 强制管理员权限运行

---

# 🚀 九、设计哲学总结

Dateview 的最终定位：

> 不是一个“日历应用”，而是 Windows 任务栏的时间感知增强层
